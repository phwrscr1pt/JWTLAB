JWT Hacking Workshop

Learn JWT vulnerabilities through hands-on practice

FOR EDUCATIONAL PURPOSES ONLY

This application contains intentional vulnerabilities. Never deploy in production.

1 Unverified Signature Easy

The server decodes JWT tokens but completely skips signature verification. Modify the payload to escalate your privileges from "user" to "admin".

Start Lab 1

2 None Algorithm Attack Easy

The server accepts the "none" algorithm, which means no signature is required. Change the algorithm header and remove the signature to forge tokens.

Start Lab 2

3 Weak Secret Key Medium

The server uses HS256 with a weak, easily guessable secret key. Brute-force the secret and sign your own admin token.

Start Lab 3

4 KID Path Traversal Medium

The server reads the key file path from the "kid" header without validation. Exploit path traversal to use a file with known content as the secret.

Start Lab 4

5 Algorithm Confusion (RS256 to HS256) Hard

The server expects RS256 but also accepts HS256, using the public key as the HMAC secret. Exploit this confusion to sign tokens with the publicly available key.

Start Lab 5
